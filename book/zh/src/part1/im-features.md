# 核心 IM 功能一览

> **定位**：本章速览 Robrix2 作为 IM 的功能清单。前置依赖：第 1 章。它们同时是第二部分所有协作能力的地基 —— 每一项后文都会派上用场。

## 会话与消息

- **房间 / 私聊 / 邀请**：左侧统一列出 Rooms、People、Invites；收到邀请可直接 Join / Reject，无需重启即时生效。
- **Thread（消息线索）**：任何消息可展开为独立 Thread，Thread 在自己的标签页中打开，也可在右侧 Threads 面板总览一个房间的全部线索。第 5.3 章整章建立在这个功能上。
- **富文本与 Markdown**：输入框原生支持 Markdown；消息渲染 HTML 格式（`formatted_body`），代码块、列表、加粗一应俱全 —— Agent 的结构化汇报全靠它。
- **@ 提及**：输入 `@` 弹出房间成员选择器，人与机器人账号混合补全；被提及的消息高亮提醒。
- **斜杠命令**：输入 `/` 弹出命令面板 —— 房间管理命令、机器人命令，以及第 5.2 章介绍的 workflow 命令。
- **回复 / 编辑 / 转发 / 删除**：完整的消息操作集，编辑与回复关系严格遵循 Matrix 规范。
- **附件与媒体**：图片、文件收发，图片预览，链接卡片预览。

## 安全与账户

- **端到端加密（E2EE）**：加密房间开箱即用，基于 matrix-rust-sdk 的 Megolm/Olm 实现；设备管理在 Settings → Devices。第 5.4 章的审批私聊完全依赖这一层。
- **多语言界面**：内置 i18n（英文 / 简体中文）。
- **实时翻译（Labs）**：可配置一个 OpenAI 兼容 API，在输入栏内实时翻译消息 —— 跨语言团队（或跨语言的人机协作）直接受益。

## 工作区

- **多标签 Dock**：房间、Thread、审批私聊各占一个标签，同屏铺开多个协作现场；
- **全局搜索**、**未读徽标**、**桌面通知**。

这些 IM 能力不是 agent-chat 的授权层：搜索结果、徽标、标签页和通知都只帮助人观察。房间成员关系与事件发送者来自 Matrix，审批是否有效则由 agent-chat 服务端决定。
