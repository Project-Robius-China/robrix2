# 前言：什么是 HAgency

> **定位**：本章回答「HAgency 是什么、为什么存在」，并给出全书的阅读方式。无前置依赖，所有读者从这里开始。

**HAgency = Human + Agency。**

AI 编码智能体（Claude Code、Codex……）已经能独立完成大量工程工作。但大多数「多智能体」产品把人挤到了边缘：人变成一个发起提示词的按钮，剩下的过程既看不见，也插不上手。

HAgency 想要另一种形态：**在人与 Agent 共存的世界里，人依然是具备能动性（Agency）的主体**。人和智能体在同一个空间里对话、分工、争论、汇报；关键决策由人拍板，高危操作由人授权；Agent 团队自治运转，但始终对人透明、可被随时介入。

这不是一个单独的产品，而是三个开源项目组合出来的一套协作系统：

| 项目 | 角色 |
|------|------|
| [Robrix2](https://github.com/Project-Robius-China/robrix2) | Rust + Makepad 编写的原生 Matrix 客户端 —— 人的工作台 |
| [agent-chat](https://github.com/ZhangHanDong/agent-chat) | 本地优先的 Agent 协调系统 —— Agent 的调度中枢与 Matrix 桥 |
| [Palpo](https://github.com/palpo-im/palpo) | Rust 编写的 Matrix homeserver —— 中立的通信底座 |

三者以 **Matrix 协议**为公共语言。选择 Matrix 不是偶然：

- **开放协议**：任何人的 agent-chat 实例、任何 Matrix 客户端都能接入同一空间 —— 本书截图中就有两个人各自的 Agent 团队在同一房间协作的实例；空间跑在你自己的服务器上，并可进一步与整个 Matrix 联邦网络互通；
- **端到端加密**：人对 Agent 的授权通道（审批私聊）受 E2EE 保护，连服务器都读不到审批内容；
- **中立底座**：人用 Robrix2（或任何 Matrix 客户端）参与，Agent 通过桥接木偶账号参与 —— 双方在协议层完全平等。

## 阅读准备

### 前置知识

- **需要**：基本的命令行操作；知道 Git 与 GitHub 的日常用法；
- **有更好**：用过任意 Matrix 客户端（Element 等）；用过 Claude Code 或 Codex CLI；
- **不需要**：Rust 或 Makepad 开发经验（除非你想改 Robrix2 本身）；Matrix 协议细节。

### 推荐阅读路径

**路径 A：我想尽快跑起来**（使用者）

> 前言 → 第 4 章部署指南（选一条路线）→ 第 5 章团队协作实战（按截图操作）

**路径 B：我想先搞清楚它凭什么可信**（评估者 / 安全视角）

> 前言 → 第 3 章理念与架构 → 第 5.4 章 Owner 审批 → 第 6 章安全模型 → 再回到部署

### 版本基线

本书基于 2026 年 7 月的三个仓库主线版本写作：Robrix2 1.1.x、agent-chat 当前主线、Palpo 当前主线。这套系统处于活跃开发中 —— 命令与界面可能演进，但第 3 章的架构与第 6 章的安全模型是长期承诺，变化最慢。书中所有截图均来自真实运行中的系统，未做摆拍。

## 本书结构

**第一部分**（第 1–2 章）简要介绍 Robrix2 作为 Matrix IM 客户端本身 —— 即使不接入任何 Agent，它也是一个完整可用的即时通讯工具。

**第二部分**（第 3–6 章）是本书重点：理念与架构、两条部署路线、五个截图驱动的协作场景（请进 Agent、作战室、Thread、审批、四角色工作流），以及支撑这一切的安全模型。
